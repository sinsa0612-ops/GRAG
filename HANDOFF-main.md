# HANDOFF-main.md (마일스톤 / 확정 아키텍처)

> 인간 주도권 문서. **사용자가 최종 확정·승인한 핵심 마일스톤과 검증된 아키텍처**만 기록한다.
> AI는 사용자의 명시적 승인 없이 이 파일의 마일스톤을 임의로 수정·조작하지 않는다.
> (본 파일은 2026-06-29 세션 종료 시 AI가 최초 생성했다. 마일스톤 표현은 사용자가 검토·조정 가능.)

## 확정 아키텍처 원칙
- **3중 DB**: SQLite(마스터/문서·해시·사용량·계층) + ChromaDB(벡터) + KùzuDB(그래프). 접근은 DAL(`db/*`)·어댑터(`adapters/*`)로만.
- **컬렉션 격벽**: 엔티티 정체성 `(collection, name)`. 사업끼리 자동으로 안 섞임. `--all`은 행정 종합.
- **추출 = 가성비**: 무료 Gemini Flash-lite + 청크당 1회. 품질은 프롬프트·청킹으로만. LangChain 전면도입 안 함.
- **유기적 연결 = 옵트인**: 격벽을 깨지 않고, 명시적 SAME_AS 브릿지로만 사업 간 연결. 계층은 부모-자손 메타.

## 마일스톤
- **M0 (확정):** 초기 GraphRAG — 컬렉션 격벽 + `graphrag` 통합 CLT. 커밋 `8c7d945`.
- **M1 (확정):** 가성비 추출 품질·정합성 보강(문장경계 청킹, predicate 정규화, 집계·격벽·백업·한도 정합). 커밋 `d3b4e51`.
- **M2 (구현·검증 완료, 사용자 커밋으로 확정 예정):** 컬렉션 유기적 고도화 ①SAME_AS 브릿지 ②교차 인사이트 질의 ③컬렉션 계층. `pytest` 128 passed. 상세는 `HANDOFF-sub.md`.
