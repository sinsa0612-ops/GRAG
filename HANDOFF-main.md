# HANDOFF-main.md (마일스톤 / 확정 아키텍처)

> 인간 주도권 문서. **사용자가 최종 확정·승인한 핵심 마일스톤과 검증된 아키텍처**만 기록한다.
> AI는 사용자의 명시적 승인 없이 이 파일의 마일스톤을 임의로 수정·조작하지 않는다.
> (최초 생성 2026-06-29. 최근 갱신 2026-07-03 — 사용자 승인 하에 M5 retrieval 개선 반영.)

## 확정 아키텍처 원칙
- **3중 DB**: SQLite(마스터/문서·해시·사용량·계층) + ChromaDB(벡터) + KùzuDB(그래프). 접근은 DAL(`db/*`)·어댑터(`adapters/*`)로만.
- **컬렉션 격벽**: 엔티티 정체성 `(collection, name)`. 사업끼리 자동으로 안 섞임. `--all`은 행정 종합.
- **추출 = 가성비 기본, 확장은 옵트인**: 기본은 무료 Flash-lite·청크당 1회, 품질은 프롬프트·청킹으로. 더 높은 recall이 필요하면 **모델 토글**(Gemma, RPD 1.5k·단 느림)과 **gleaning**(`--glean N`, 청크당 다회독)을 옵트인으로만 켠다. LangChain 전면도입은 여전히 안 함(외부 기법은 아이디어만 경량 자체 구현).
- **유기적 연결 = 옵트인**: 격벽을 깨지 않고, 명시적 SAME_AS 브릿지로만 사업 간 연결. 계층은 부모-자손 메타.
- **품질 검증 = 상대 비교**: 추출/모델/프롬프트 변형의 우열은 `graphrag eval`(질문 자동생성 + LLM 페어와이즈 심판)로 '답변 품질'을 잰다(엔티티 '수' 프록시가 아님).

## 마일스톤
- **M0 (확정):** 초기 GraphRAG — 컬렉션 격벽 + `graphrag` 통합 CLI. 커밋 `8c7d945`.
- **M1 (확정):** 가성비 추출 품질·정합성 보강(문장경계 청킹, predicate 정규화, 집계·격벽·백업·한도 정합). 커밋 `d3b4e51`.
- **M2 (확정):** 컬렉션 유기적 고도화 ①SAME_AS 브릿지 ②교차 인사이트 질의 ③컬렉션 계층. `pytest` 128 passed. 커밋 `6c47b84`.
- **M3 (확정):** 추출 품질 개선 Tier 1~3 — 프롬프트 잡티 차단, 표기 정규화 병합(무료), 표 노이즈 전처리(청크 378→191). 커밋 `7e95da9`.
- **M4 (확정):** 명령 통합 GUI(7탭) + 추출 모델 토글(Gemma) + gleaning(옵트인) + 품질평가 하네스(`graphrag eval`). `pytest` 145 passed. 커밋 `ffa2ea5`. 상세는 `HANDOFF-sub.md`·`walkthrough.md`.
  - 검증된 발견: gleaning 1라운드 recall +45%(flash·gemma 공통). gemma는 API 지연이 커(콜당 75~130초) 대형 문서엔 배치용. `eval` 첫 실측(단순 vs 풍부 그래프)에서 풍부한 추출이 사실 질문엔 우세하나 상당수 무승부 → 다음 병목은 추출량이 아니라 retrieval/답변 합성일 수 있음.

- **M5 (확정):** retrieval/답변 합성 개선. ①`top_k` 설정화(8→12) + **벡터→그래프 브릿지**(질문에 엔티티 이름이 없어도 벡터로 찾은 본문의 엔티티를 그래프로 표면화 — 잠수함 A/B 개선후 4승·무3·회귀0, 그래프 표면화 엔티티 1→20). ②**답변 합성 충실성 재구성**(본문=1차 근거·그래프=보조, 충돌 시 본문 우선·미검증 그래프 주장 단정 금지 — 캐럴 flash-g1·gemma-g1 전/후 eval 신 8승·0패·8무, 그래프발 환각 제거). 커밋 `574f030`·`53e9065`·`5578d08`·`a53f2e4`. `pytest` 151 passed.
  - 검증된 발견(공정 4자 재적재 비교 flash/gemma×glean0/1, 벡터+그래프 정상 적재): 벡터 검색이 제대로 있으면 **추출 recall 2배 차이(gemma-g1 660 vs flash-g0 296 엔티티)도 Q&A 답변엔 거의 전환 안 됨** → 레버리지는 추출이 아니라 retrieval/합성. (M4의 'retrieval 병목' 방향성 확정. 단 M4 eval 수치는 벡터 미적재 캐럴 컬렉션 탓 무효였고 M5 공정 재적재로 재확인.) 서사(캐럴)·표/희소(잠수함) 문서 양쪽 답변 양호 → **retrieval은 견고한 상태로 종료.**
  - 부수: gemma 스레드풀 병렬 정상(프로브 6콜=5.1x; 느려 보임은 콜당 ~100초 채움구간 착시). flash-lite 하드 한도=500/일(API 명시).
